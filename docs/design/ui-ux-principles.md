# UI/UX Design Principles

## 적용된 UX 법칙

### 1. 제이콥의 법칙 (Jakob's Law)
> 사용자는 대부분의 시간을 다른 사이트에서 보낸다

- **적용**: Product Hunt, Futurepedia 등 기존 AI 디렉토리 패턴 참조
- **구현**: 카드 기반 레이아웃, 좌측 카테고리, 상단 검색바 (검색 컨벤션)
- **적용 페이지**: 홈, 카테고리, 검색

### 2. 피츠의 법칙 (Fitts's Law)
> 타겟이 크고 가까울수록 쉽게 클릭

- **적용**: 홈페이지 검색바를 크게 (size="lg"), CTA 버튼 충분한 패딩
- **구현**: "Visit Website" 버튼 px-6 py-3, 카드 전체가 클릭 영역
- **적용 페이지**: 홈 Hero, 도구 상세

### 3. 밀러의 법칙 (Miller's Law)
> 인간은 7(+-2)개 항목을 기억

- **적용**: 카테고리 10개 (관리 가능한 수), 홈에 Featured 6개
- **구현**: 그리드 최대 3열, 카테고리 명확한 그룹핑
- **적용 페이지**: 홈, 카테고리 목록

### 4. 힉의 법칙 (Hick's Law)
> 선택지가 많을수록 결정 시간 증가

- **적용**: 카테고리로 선택지 분류, 필터로 결과 축소
- **구현**: 가격 필터 4개만 (All/Free/Freemium/Paid), 정렬 3가지
- **적용 페이지**: 검색, 카테고리

### 5. 포스텔의 법칙 (Postel's Law)
> 보내는 것은 엄격하게, 받는 것은 관대하게

- **적용**: 검색어 처리 (LIKE %query% 패턴), URL slug 유연성
- **구현**: 대소문자 구분 없는 검색, 부분 매칭 지원

### 6. 피크엔드의 법칙 (Peak-End Rule)
> 경험의 정점과 끝이 전체 인상을 결정

- **적용**: 도구 상세 페이지의 "Visit Website" CTA (피크), 관련 도구 추천 (끝)
- **구현**: gradient 버튼으로 시각적 피크, 사이드바 관련 도구로 자연스러운 이탈

### 7. 심미적 사용성 효과 (Aesthetic-Usability Effect)
> 아름다운 디자인이 더 사용하기 쉽다고 인식

- **적용**: 깔끔한 카드 UI, 그라데이션 아이콘, 부드러운 hover 애니메이션
- **구현**: rounded-xl 카드, 그라데이션 로고, shadow-sm, hover:-translate-y-1

### 8. 폰 레스토프 효과 (Von Restorff Effect)
> 차별화된 항목이 더 잘 기억됨

- **적용**: Featured 도구에 골드 배지, 가격 유형별 컬러 코딩
- **구현**: amber-400 Featured 배지, green/blue/purple 가격 뱃지

### 9. 테슬러의 법칙 (Tesler's Law)
> 복잡성은 줄일 수 없고 이동만 가능

- **적용**: 복잡한 필터 로직은 서버에서 처리, 사용자에게 단순한 UI 제공
- **구현**: URL 파라미터 기반 필터링, 서버 사이드 쿼리

### 10. 도허티 임계 (Doherty Threshold)
> 응답 시간 400ms 이하면 생산성 급증

- **적용**: Next.js SSR + SQLite 로컬 쿼리 → ms 단위 응답
- **구현**: 서버 컴포넌트 기본 사용, 클라이언트 JS 최소화 (102KB shared)

## 모바일 우선 반응형 설계
- 모든 그리드: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- 네비게이션: 모바일에서 검색바만 표시 (md:flex로 메뉴 숨김)
- 카드: 모바일에서 풀 폭, 태블릿 2열, 데스크탑 3열

## 접근성
- 시맨틱 HTML (nav, main, h1-h3, a)
- aria-hidden on ad slots
- 충분한 색상 대비 (gray-900 on white)
- 키보드 네비게이션 지원 (a 태그 기반)
